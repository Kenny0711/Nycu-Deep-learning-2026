# Online Reinforcement Learning in Non-Stationary Context-Driven Environments (LCPO)

> Hamadanian et al., MIT CSAIL / Brown / Microsoft Research — **ICLR 2025**

## 一句話

在環境隨時間改變的線上 RL 中，不用 task label、只靠 OOD 偵測來「錨定」舊 context 的 policy 輸出，從而對抗 Catastrophic Forgetting。

---

## 動機

### 問題：非穩態環境下的 Catastrophic Forgetting（CF）

線上 RL（online RL）要求 agent 在 **真實環境中邊跑邊學**，不像傳統 RL 先在模擬器裡訓練好再部署。真實環境的動態往往隨時間改變——例如網路壅塞程度、機器人受到的外力、使用者請求速率。

**Context-driven MDP**：非穩態的來源是一個外生的 context 過程 $\mathbf{z} = \{z_t\}_{t=1}^\infty$。Agent 觀測到 $z_t$，但不能預知未來的 $z$。

當 context 改變，基於神經網路的 on-policy 演算法（A2C, TRPO 等）會**忘掉過去學的東西**。數學直覺：neural network 是「共用一套參數」去表示所有 $(s, z)$ 的 policy，所以更新目前 context 下的參數，**必然干擾其他 context 下的輸出**，這就是 CF。

### 為什麼現有方法不夠好？

| 類別 | 代表方法 | 問題 |
|------|----------|------|
| Task-based | EWC, OGD, Progressive NN | 需要 **task label**，現實中往往不可得 |
| Regularization heuristic | Sliding OGD, BFDQN | 假設每個 episode 是獨立 task，脆弱 |
| Off-policy (rehearsal) | SAC, CLEAR, DDQN | 超參數敏感、訓練不穩定；在線上設定表現差 |
| Task inference (CPD) | MBCD | 需要 context 有清楚分段邊界，不現實 |

**關鍵洞見**：如果 policy 能區分不同 context（它確實可以，因為 $z_t$ 是觀測量），那只要確保更新時「對其他 context 的輸出保持不動」就夠了——**不需要知道哪些 context 是什麼任務**，只需要知道哪些樣本來自「跟現在不同」的 context（OOD 偵測）。

## 方法

### 形式化設定

**Non-stationary context-driven MDP**：$\mathcal{M} = (\mathcal{S}, \mathcal{Z}, \mathcal{A}, \{z_t\}_{t=1}^\infty, T, d_0, r)$

- $s_t \in \mathcal{S}$：環境狀態
- $z_t \in \mathcal{Z}$：外生 context（agent 可觀測，但不影響 context 本身）
- $a_t = \pi(s_t, z_t)$：policy 輸出
- $r_t = r(s_t, z_t, a_t)$：reward
- $T(s_{t+1} | s_t, z_t, a_t)$：轉移函數

**目標**：最大化 lifelong return $J(\pi, \mathbf{z}) = \lim_{t \to \infty} \sum_{i=1}^t \frac{R_i}{t}$。

> 最佳 policy 需要「先知」（prescient）——提前知道整個 context 軌跡。線上 agent 只能用到當前為止的 context，因此跟 prescient agent 之間必然有 gap。LCPO 的目標是**最小化這個 gap**。

### 直覺：類比 Tabular RL

在 tabular RL（有限 state/action/context）中，每個 $(s, z)$ 對有獨立的表格 entry。更新某個 $(s, z_t)$ 的 Q 值，**完全不影響** $(s, z' \neq z_t)$ 的 Q 值。這就是為什麼 tabular RL 在這個設定下不會有 CF。

**能不能用 neural network 模擬 tabular 的行為？**

只要在更新目前 context 的參數時，**額外加一個約束**：要求 neural network 在「其他 context 的樣本上輸出不變」就行了。

### LCPO 核心：Constrained Policy Optimization

設 $B_r$ 為最近一批新經驗（current context），$B_a$ 為歷史所有經驗的 buffer。OOD sampler $W(B_a, B_r)$ 從 $B_a$ 中取出 context 跟目前明顯不同的樣本（稱作 **anchoring samples** $S_c$）。

**LCPO 的優化問題**：

$$\min_\theta \mathcal{L}_\text{tot}(\theta; B_r) \triangleq \mathcal{L}_\text{PG}(\theta; B_r) + \mathcal{L}_e(\theta; B_r)$$

$$\text{s.t.} \quad D_\text{KL}(\theta_0, \theta; W(B_a, B_r)) \leq c_\text{anchor} \tag{1}$$

逐項解釋：
- $\mathcal{L}_\text{PG}(\theta; B_r) = \mathbb{E}_{r_t \sim B_r}\left[\sum_{t=0}^H -\gamma^t r_t\right]$：標準 policy gradient loss，最大化當前 batch 的 return
- $\mathcal{L}_e(\theta; B_r) = e^{\theta_e} \mathbb{E}_{s_t, z_t \sim B_r, a_t \sim \pi}[\log \pi(a_t|s_t, z_t)]$：自動熵正規化（entropy regularization），促進探索
- **約束**：$D_\text{KL}(\theta_0, \theta; W(B_a, B_r)) \leq c_\text{anchor}$，即在 OOD 樣本上 KL divergence 不能太大，等價於「不要改變對其他 context 的 policy 輸出」

> 為什麼不用 ground truth action anchoring？因為過去 policy 未必收斂，若直接用過去採取的 action 當 anchor，等於強迫重學 suboptimal behavior。KL 約束只保證輸出「不變」，而非「對齊某個特定答案」，更安全。

### 求解方法（類 TRPO）

跟 [[TRPO]] 結構類似，用一階近似目標、二階近似約束：

$$\min_\theta (\theta - \theta_0)^T v \quad \text{s.t.} \quad (\theta - \theta_0)^T A (\theta - \theta_0) \leq c_\text{anchor} \tag{4}$$

其中：
- $v = \nabla_\theta \mathcal{L}_\text{tot}(\theta; \cdot)|_{\theta_0}$：一階梯度（policy gradient 方向）
- $A_{ij} = \frac{\partial}{\partial \theta_i}\frac{\partial}{\partial \theta_j} D_\text{KL}(\theta_0, \theta; W(B_a, B_r))$：Fisher Information Matrix（在 OOD 樣本上）

求解方式：**Conjugate Gradient + Line Search**（和 TRPO 相同）：
1. 用 conjgrad 求出滿足約束的搜索方向 $v_c$
2. Line search：若 $\theta_\text{old} + v_c$ 違反任何約束（anchoring 或 TRPO-style recent constraint）則 $v_c \leftarrow v_c / 2$，直到滿足

### 三個關鍵組件

#### 1. Buffer Management — Reservoir Sampling

**問題**：$B_a$ 會無限增長。

**解法**：Reservoir Sampling（Vitter, 1985）——維護大小為 $n_b$ 的 buffer，每個新樣本以 $\frac{n_b}{n_s}$ 機率替換舊樣本（$n_s$ = 目前總樣本數）。數學保證：buffer 中的樣本是所有歷史樣本的**均勻隨機子集**。

實驗中設 $n_b = $ 1% of all samples（最多 200K）。

#### 2. OOD 偵測 — W(Ba, Br)

判斷 $z_i$ 是否跟目前 $B_r$ 的 context 夠不同（OOD）：

- **L2 thresholding**（gym 風力環境）：
  $$W(B_a, B_r) = \{w_i \in B_a : \|w_i - \bar{w}_r\|_2 > \sigma\}$$
  其中 $\bar{w}_r = \mathbb{E}_{w \sim B_r}[w]$ 是最近 batch 的平均風力向量。

- **Mahalanobis Distance**（straggler mitigation 環境）：
  $$\omega(w, B_r) = (w - \mu_w)^T \Sigma_w^{-1} (w - \mu_w) > \sigma^2$$
  用高斯模型 fit $B_r$，取距離超過閾值的樣本。

**OOD sampling 效率**：從 $B_a$ 取 $s = 5b$ 個候選樣本，留下 OOD 的；若不足 $b$ 個則跳過該次約束更新。

#### CPD vs. OOD 偵測的本質差異

Change-Point Detection 要求 context 有清楚的分段邊界（piecewise stationary）。OOD 只需要定義「距離」函數——對任意 context process 都有意義。

> 對一個平滑連續的 context trace（如 Ornstein-Uhlenbeck 過程），CPD 在 σ=3.1 時偵測到 0 個切換點，在 σ=3.0 時偵測到 34 個（多數無意義）。OOD 只需判斷 $|z_i - z_j| > 1$，直覺清晰。

## 理論保證

### Theorem B.2（Tabular Context-Driven RL 的最優性）

**條件**：
1. Rewards 有界：$|r_t| \leq R$
2. 學習率 $\{\alpha_i\}$ 滿足 Robbins-Monro 條件：$\sum \alpha_i = \infty$，$\sum \alpha_i^2 < \infty$
3. 狀態、動作、context 空間有限：$|\mathcal{S}|, |\mathcal{A}|, |\mathcal{Z}| < \infty$
4. 任意連續 $N$ 個 episode 中，每個 $z \in \mathcal{Z}$ 至少出現一次
5. Context 只在 episode 邊界改變

**結論**：Q-learning 收斂到最佳 policy。

**直覺**：Context 只在 episode 邊界切換 → 同一 episode 內的 Q-learning 更新只涉及單一 context $z$，不同 context 的 Q-value 更新完全解耦。再用 Lemma B.1 證明每個 context 的學習率子序列也滿足 Robbins-Monro 條件，因此每個 context 的 Q-learning 獨立收斂。

**關鍵 Lemma**（B.1）：若 $\{\alpha_i\}$ 滿足 Robbins-Monro 條件，它的任何子序列（每 N 個取一個）也滿足該條件。

這個定理證明了 tabular setting 下的最優性，也為 LCPO 的設計提供了理論直覺。

## 實驗

### 環境設定

6 個環境（均含外生 context 過程）：

| 環境 | Context 形式 |
|------|-------------|
| Pendulum-v1（改） | 外部風力（Ornstein-Uhlenbeck / piecewise Gaussian） |
| InvertedPendulum-v4 | 外部風力 |
| InvertedDoublePendulum-v4 | 外部風力 |
| Hopper-v4 | 外部風力 |
| Reacher-v4 | 外部風力 |
| Straggler Mitigation | 真實 Microsoft 雲端 workload traces（2018 年 2 月） |

Context traces：4 種合成序列（smooth, sudden, stochastic, predictable）。Agent 觀測過去 3 個時間步的風力向量作為 context（因為 context 作用在 next step）。

**評分指標**：Normalized Return（0 = 所有 agent 最差，1 = 最好）。

**Baselines 共 11 種**（含 prescient agent）：EWC, Sliding OGD, BFDQN, MBCD, MBPO, CLEAR, PT-DQN, SAC, DDQN, A2C, TRPO。

### 主要結果（§5.1）

**LCPO 在所有 online agents 中表現最好，且最接近 prescient agent。**

- **A2C**（最強的 on-policy baseline）能快速 adapt，但無法維持對舊 context 的記憶
- **SAC / DDQN**（off-policy）：訓練不穩定，常跑輸 A2C
- **MBCD**：頻繁誤判 task boundary（不同 seed 偵測到 3~7 個切換點）
- **CLEAR, PT-DQN**：在調過參數的 Pendulum-v1 上尚可，但在其他環境崩潰
- **MBPO**：即使用 perfect world model，也因為 buffer sampling 偏向早期 context 而表現差
- **Online EWC, Sliding OGD**：隱含假設每個 episode 是獨立 task，在實際問題中失敗

計算開銷：LCPO 約為 A2C 的 **1.5×**，遠低於 MBCD（11×）和 MBPO（4.2×）。

### Straggler Mitigation（真實環境）

| 演算法 | Workload 1 (tail latency ms) | Workload 2 |
|--------|------------------------------|------------|
| **LCPO Cons** | **1048 ± 7** | **586 ± 27** |
| A2C | 1716 ± 710 | 604 ± 109 |
| DDQN | 1701 ± 47 | 633 ± 7 |
| Best Prescient | 984 (TRPO) | 509 (A2C) |

LCPO 顯著優於其他 online methods，且接近 prescient bound。

### OOD 閾值敏感度（§5.2）

LCPO 對 $\sigma$ 有一定敏感度，但在所有測試閾值下均優於 A2C baseline。**Mahalanobis distance 比 L2 更穩健**（即使在不能分離 state 與 context 的情況下）。

### Buffer 大小敏感度（§5.3）

- $n_b \geq 500$：效果幾乎不受影響
- $n_b < 500$：效果顯著下降
- 原因：context trace 在短時間內變化不劇烈，500 個隨機樣本足以代表整個 trace 的分布

## LCPO-P 變體（Appendix E）

**想法**：直接把 anchoring 加到 loss 裡（PPO 框架）：

$$\mathcal{L}_\text{anchor}(\theta, \theta_0; B_r, B_a) = \mathbb{E}_{s,z \sim W(B_a, B_r)}[\text{CELoss}(\pi_{\theta_0}(s,z), \pi_\theta(s,z))]$$

$$\min_\theta \mathcal{L}_\text{PG}(\theta; \cdot) + \kappa \cdot \mathcal{L}_\text{anchor}(\theta, \theta_0; B_r, B_a)$$

**問題**：$\kappa$ 需要對每個環境仔細調整，且在 $\theta = \theta_0$ 時梯度為 0，需要多次 gradient step 才生效。實驗顯示 LCPO-P 在 Pendulum-v1 上未能超過最佳 baseline。

> 這說明 Lagrangian dual 方法（LCPO-P）和 primal constrained 方法（LCPO）在數學上等價，但實際上需要 $\kappa$ 精確對應每個 gradient step 的 dual variable，否則效果大打折扣。

## Discussion & Limitations

**Network Capacity**：Neural network 有有限容量，學太多 context 後可能達到容量上限。若發生這種情況，可考慮從 $B_a$ 中移除過舊的樣本，讓 agent 選擇「遺忘遠古 context」以換取靈活性。

**Exploration**：LCPO 用 automatic entropy tuning 做探索，但這並非專為 non-stationary 設計的。當 context 突然改變，agent 需要快速探索新 context——此處可結合 curiosity-driven exploration。

**Buffer 策略**：目前用 Reservoir Sampling（均勻隨機），未來可考慮 context-aware buffer（例如最大化 context diversity）。

**適用條件**：
1. 非穩態必須由**可觀測的外生 context** 驅動
2. 需要一個能區分「不同 context」的 **OOD 相似度指標**

比起需要 task label 的方法，這兩個條件更容易滿足。

## 思考題

1. **和 TRPO 的關係**：LCPO 的約束方向剛好和 TRPO 相反——TRPO 約束「目前 context 上 policy 不能改太多」，LCPO 約束「其他 context 上 policy 不能改」。若把兩個約束都加，是否能進一步提升穩定性？（論文確實同時做了兩個：主要用 conjgrad 解 LCPO 約束，再用 line search 確保 TRPO 約束。）

2. **OOD 閾值的自動調整**：論文指出 OOD 閾值 $\sigma$ 沒有做超參搜索。在 warm-up 期間（前 6M steps），能否用 MDP 轉移的 surprise 訊號自動校準 $\sigma$？

3. **連接到 DLP 課程**：DLP 課程中的 Policy Gradient（REINFORCE, A2C, PPO）都假設環境是穩態的。LCPO 的核心貢獻是在 policy gradient 框架下加入「對過去 context 的記憶保護」——相當於把 continual learning 的 rehearsal 思想移植到 on-policy RL，但避免 off-policy 帶來的不穩定性。

4. **與 Meta-RL 的邊界**：LCPO 假設 context 可觀測，而 Meta-RL 假設 context 未知需要推斷。如果 context 是潛變數（latent），先用 context inference 估計 $\hat{z}_t$，再套用 LCPO，理論上可行，論文也提到這是 future work。
