## 1. Value-Based RL 核心概念

| 概念 | 說明 |
|---|---|
| **Q-function** `Q(s,a)` | 在狀態 s 採取動作 a 後，預期累積 reward |
| **Bellman Equation** | `Q(s,a) = r + γ · max_a' Q(s',a')` |
| **Policy** | 從 Q-function 導出：`π(s) = argmax_a Q(s,a)` |
| **ε-greedy** | 以 ε 機率 random explore，1-ε 機率 exploit |

---

## 2. Vanilla DQN

### 核心設計
1. **神經網路** 近似 Q-function：`Q(s, a; θ)`
2. **Experience Replay**：把 transition `(s, a, r, s')` 存進 Replay Buffer，隨機 sample → 打破 correlation
3. **Target Network**：獨立的 `Q(s', a'; θ̄)`，每 N 步才從 main network 複製 → 穩定訓練目標

### Loss Function
$$
L(\theta) = \frac{1}{2} \sum_{(s,a,r,s') \in D} \left( r + \gamma \max_{a'} Q(s', a'; \bar{\theta}) - Q(s, a; \theta) \right)^2
$$

> **Bellman Error** = `r + γ·max Q(s',a'; θ̄) - Q(s,a; θ)`
> → 這就是 TD error，loss 是其平方的均值

---

## 3. Double DQN (DDQN)

### 問題：Vanilla DQN 高估 Q 值
- `max Q(s', a'; θ̄)` 用同一個 network 選動作 **又** 評估值 → 系統性高估

### 解法：解耦「選動作」和「評估值」
$$
\text{Target} = r + \gamma \cdot Q\!\left(s', \arg\max_{a'} Q(s', a'; \theta);\; \bar{\theta}\right)
$$

- 用 **main network θ** 選出最好的 action
- 用 **target network θ̄** 評估這個 action 的值

### 實作差異（只改一行！）
```python
# Vanilla DQN
next_q = target_net(next_states).max(1)[0]

# Double DQN
best_actions = main_net(next_states).argmax(1)
next_q = target_net(next_states).gather(1, best_actions.unsqueeze(1)).squeeze(1)
```

## 4. Prioritized Experience Replay (PER)

### 問題：Uniform Sampling 效率低
- 不是所有 transition 學習價值相同，「驚訝」的 transition（TD error 大）更值得多學

### Priority 定義
$$
p_i = |\delta_i| + \epsilon
$$
其中 $\delta_i = r + \gamma \max_{a'} Q(s',a') - Q(s_i, a_i)$ 是 Bellman error，$\epsilon > 0$ 保證不為零

### Sampling 機率
$$
P(i) = \frac{p_i^\alpha}{\sum_k p_k^\alpha}
$$
- $\alpha \in [0,1]$：控制 prioritization 強度，$\alpha=0$ = uniform

### Importance Sampling (IS) 修正偏差
$$
w_i = \left(\frac{1}{N \cdot P(i)}\right)^\beta
$$
- $\beta \in [0,1]$：控制修正強度，訓練後期通常退火到 1
- 更新時：loss 乘上 $w_i$（通常對 max weight 正規化）

### 實作重點
- 用 **Sum Tree** 資料結構高效 O(log n) 採樣
- 每次更新後要更新 priority：`buffer.update_priorities(indices, new_td_errors)`

## 5. Multi-Step Return

### 問題：1-step return 偏差大
- 單步 reward 後就用 bootstrap → Q 值更新緩慢，bias 高

### n-step Return
$$
R_t^{(n)} = \sum_{k=0}^{n-1} \gamma^k r_{t+k} + \gamma^n \max_{a'} Q(s_{t+n}, a')
$$

- 展開 n 步真實 reward，再加上第 n+1 步的 bootstrap
- n 越大 → variance 越高，但 bias 越低

### 實作重點
```python
# 在 replay buffer 中用 deque 維護 n-step buffer
n_step_buffer = deque(maxlen=n_steps)

# 計算 n-step return
def get_n_step_info(n_step_buffer, gamma):
    reward = 0
    for i, (r, done) in enumerate([(t[2], t[4]) for t in n_step_buffer]):
        reward += (gamma ** i) * r
        if done:
            break
    return reward, n_step_buffer[-1][3]  # (R_n, s_{t+n})
```

## 8. 訓練實戰 Tips

| 問題 | 解法 |
|---|---|
| CartPole 不收斂 | 檢查 reward shaping、learning rate（試 1e-4 ~ 1e-3） |
| Pong Q 值爆炸 | 加 gradient clipping：`nn.utils.clip_grad_norm_(params, 10)` |
| PER 實作 bug | 先確認 priority 更新時機（update 之後更新，不是 sample 之後） |
| Memory 爆掉 | Atari replay buffer size 50k~500k，視 GPU 記憶體調整 |
| 訓練太慢 | 確認用 GPU、batch size 32，frame stack 用 `LazyFrames` |
